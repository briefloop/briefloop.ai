BriefLoop.ai / 运作原理

AI 起草，控制面记账，人类放行

首页的保管链演示回答“这句话从哪来”。这一页回答其余的问题：为什么要这样分权、发现问题以后会怎样、失败如何变成下一版的改进，以及开发者如何与 Agent 协作。

Governance · 治理模型

给 AI 装一套「公司治理制度」

AI 负责起草与提议，BriefLoop 确定性控制层负责审计与记账。各司其职，才有可信。

传统 AI 写作（黑箱）

  • 数据真实性存疑：大模型逻辑不稳定，改了几稿后，数据真实来源往往无从复查。
  • 错误以讹传讹：草稿中的一个微小误读，在反复润色后被包装成高可信度的错误结论。
  • 反馈凭空蒸发：人工提出的偏好与规则留在脑子里，AI 在下一次运行中极易遗忘重犯。
  • 复核成本高昂：因为不了解修改和证据链路径，复核团队只能靠人力反复重核原始文档。

BriefLoop 保管链（白盒可审计）

  • 事实账本登记：每一个写进交付物的核心断言和数据，必须在事实账本 (Claim Ledger) 有坐标。
  • 确定性质量门禁：由 Python 逻辑执行时效、来源完整性门禁，防止模型自画自签。
  • 改进账本固化：人工批准的写作偏好沉淀为持久记忆，并提供防污染的回滚记录。
  • 交付真值单一记录：交付状态只认 finalize_report 与完成投影，reader-clean 未通过就不会晋升交付件——文件存在不等于交付。

Evolution Loop · 演进环

每次失败可以成为下一版的测试，但不会自动改写系统

单次运行负责生产简报；演进环负责把已记录的失败变成可审阅的改进候选。只有通过评估并经人类批准的变更，才进入下一版本。

Governance Boundary · 治理边界

人类责任边界声明

任何自动化 AI 系统都不应当、也不能代替人类承担决策与把关的责任。BriefLoop 并不代替人类进行最终的批准决策，而是为人类提供一个降低遗漏风险的证据链条——可追溯性，而非语义证明。

"一旦数据存在冲突或缺失，系统将暴露痕迹并支持快速回溯定位。"
Responsibilities · 责任分层
AI 智能体 起草、分析和提出建议 ⌄
可以找资料、拆分断言、写作和提出审计意见；不能冻结账本、通过门禁或批准交付。
确定性控制面 校验、记账、执行门禁并保留记录 ⌄
检查 schema、哈希、阶段状态、门禁结果和归档完整性；不判断来源是否代表事实真理。
人类决策者 最终批准交付并承担责任 ⌄
复核重要推论和未决问题，决定降级、返工或交付；批准动作会被记录。
Report Packs · 交付入口

支持三大交付入口

针对投研、合规、企业披露的不同任务，系统预置了高标准的拓扑与审计合同。

industry-weekly

行业 / 市场周报追踪

用于监测行业动态、竞品新闻与政策发布。抓取多源数据并进行实体坐标映射，生成可由人工每周交付的行业简报。

多源抓取 时效门禁 实体对照
⌄
典型任务

输入：近 7 天行业材料、竞品动态、政策更新与关注主题。

交付：执行摘要、关键信号、竞品变化、风险观察和来源附录。

management-monthly

管理层汇报与经营分析

支持输入公司内部财务数据与业务更新指标。在严格隔离的离线环境下建立锁定保护的事实账本 (Claim Ledger)，为决策提供可追溯的归因支撑。

指标审计 离线运行 人机分权
⌄
典型任务

输入：月度经营更新、KPI 表、部门说明和上期材料。

交付：经营亮点、环比变化、原因说明、风险与行动项，并保留审计记录包。

document-review

文档证据审阅

对大批量招股书、审计报告、监管问询函等海量文档进行检索。提取的事实精准到具体页码与高亮段落，辅助尽调分析并提供完整的审计痕迹。

段落高亮 保管链留痕 出处回溯
⌄
典型任务

输入：指定文件、锁定的审阅范围和需要回答的问题。

交付：证据摘录、来源锁定记录与可复核引用；不自动形成法律或披露结论。

Developers · 开发者

现代开发者与 Agent 协作

BriefLoop 把写作与控制分开：外部 Agent 负责研究和起草，Python CLI 负责建档、校验、状态事务、门禁与交付记录。CLI 不会单独调用模型生成简报。

确定性 CLI 交互

通过命令行轻松查看当前阶段、缺失文件、阻塞原因与审计结果报告。交付状态只认 finalize_report.json 的单一交付真值记录。

WorkBuddy / Claude Code 配合

安装对应 runtime 资产后，由 WorkBuddy 或 Claude Code 读取 operator handoff 并执行写作；Claude Code 的对话入口是 /briefloop new，不是 shell 命令。

bash — briefloop
$ pipx install briefloop
$ briefloop new industry-weekly ./my-weekly
$ briefloop run --workspace ./my-weekly --runtime operator
[created] workspace → ./my-weekly
[handoff] agent_handoff.md · agent_handoff.json
# 下一步：把 handoff 交给支持的 Agent runtime；CLI 不会自行写报告

控制面快速体验：pipx install briefloop。完整 Claude Code 路径需要源码安装并运行 briefloop claude install。
