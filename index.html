The real risk · 真正的风险

AI 写报告最危险的，往往不是胡编，而是这些「小错」

它们看起来很自然，读起来很自信，却会被直接当成最终结论发出去。

01

数字在改稿时变了

原文是「环比下降 1.5%」，改到第五版，正文里变成了「上涨 2.5%」。没人是故意的，但结论已经反了。

02

AI 自己补了一个原因

数据只显示利润率上升，AI 顺手加了一句「由于物流成本下降」——材料里根本没提到物流。

03

旧数据被写成当前情况

引用的是上一季度的份额数据，正文却写成「目前市场份额」，时间范围被悄悄抹掉。

Why not just a prompt · 为什么不只是提示词

不是更长的 Prompt，而是 AI 之外的交付规则

Prompt 和 Skill 告诉 Agent「应该怎么做」；BriefLoop 记录它「实际做了什么」，并决定哪一版有资格交付。

最大的区别不是多一个审稿 Agent，而是写稿的 Agent 没有权把自己的稿子直接宣布为「可以交付」。

负责「写」

ChatGPT + 长 Prompt

像临时请来一位聪明的分析师。

  • 阅读材料
  • 总结、分析、写稿
  • 按你的反馈继续修改
  • 按长 Prompt 尽量遵守格式和规则

但规则主要活在当前对话里。哪一版是最终稿、哪个问题已解决、数字有没有在改稿时变，通常仍靠模型自己理解和说明。

适合头脑风暴、一次性草稿、低风险内部材料。

负责「跑」

Agent Runtime + Skill.md

像给分析师配了工具、任务清单和标准操作流程。

  • 调用搜索、文件、代码等工具
  • 自动拆分并执行多步任务
  • 复用同一套 Skill、多 Agent 分工
  • 失败后自动尝试重跑

但若没有独立的状态和权限层，同一个 Agent 仍可能既写稿、又判断自己完成、又改此前的文件、又宣布「最终稿已生成」——执行者和裁判是同一个人。

适合自动化研究、信息收集、末尾人工复核的内部工作流。

负责「管住交付」

BriefLoop

不替代上面任何一个，而是包在它们外面。它增加的不是更聪明的写作，而是一层记录与权限：

  • 哪些关键事实已经确认
  • 哪个版本的工件被登记
  • 检查发现了什么、是否真的处理
  • 哪一版通过交付前检查、谁批准
  • 运行中断后应从哪里继续

这些状态不只活在模型的回答里，而是由 Agent 之外的程序和记录保存。

适合周报、月报、管理层简报、IR 材料、研究报告，以及需要多人复核、重复运行、承担交付责任的工作。

ChatGPT 负责想和写，Skill 负责教它怎么做，Runtime 负责让它跑起来，BriefLoop 负责记账、拦截和放行。

诚实说明

Agent Runtime 当然也可以自行实现持久状态、工件冻结、不可绕过的门禁、人工批准、审计日志和断点恢复。但当这些能力都被实现后，它其实已经在重建 BriefLoop 的核心。BriefLoop 的价值不是声称别人做不到，而是把每个团队原本要用 Prompt、脚本和约定各自拼出来的治理能力，做成一套现成、统一、可测试、可复用的工作流。

Skill 是「请你这样做」，BriefLoop 是「没做到这一步，就过不去」。

How it works · 怎么工作

四步，把 AI 草稿变成敢交付的最终稿

1

把资料交给 AI

本地文件、报告、表格，或已登记的网页材料。

2

AI 起草报告

可以用 Claude Code、WorkBuddy 或其他支持的 Agent。

3

BriefLoop 对照来源和规则检查

看数字有没有出处、材料是否过期、是否突然多出新的重要结论。

4

问题退回修改，人确认后交付

未处理的问题不会被当成最终稿。放行的永远是人，且留有记录。

查看技术细节

在这四步背后，每个关键事实先落进关键事实清单（claim_ledger.json）并冻结；检查结果、审计发现和放行动作写进事件日志（event_log.jsonl）；冻结时对事实做哈希，改稿时不能被悄悄改动。这些是证据，不是你每天要读的东西——完整机制见 运作原理。

Scenarios · 适用场景

你每周要交什么，BriefLoop 就帮你守住那一份

你每周要交什么？BriefLoop 帮你保留什么？最常拦住哪类问题？

行业与竞品周报

要交：每周的行业动向和竞品跟踪。

常拦住：多轮改稿后混入的未证实数据和观点；核心数字先冻结再润色。

管理层月报

要交：给管理层的月度经营与决策简报。

常拦住：把「预计」写成「已完成」、过于绝对且缺乏归因的结论。

文档证据审阅

要交：对 AI 草稿做出处与合规复核的结论。

常拦住：过期或无依据的陈述被标记并阻断，复核只看例外。

Boundary · 边界

帮你查清楚，但不替你拍板

BriefLoop 不会自动证明某句话绝对正确，也不会替代法律、合规或管理层判断。它做的是把出处、冲突、过期材料和未解决问题摆到你面前，并防止这些问题被悄悄当成最终稿。

举个例子：它不会证明「需求一定没有回暖」；它只是指出，当前提供的材料不足以支持这个结论。
