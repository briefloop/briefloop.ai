Scenarios · 典型场景

在「错了有后果」的场景里把好最后一道关

季报追问、竞品周报、合规复核、管理层简报——AI 提速的同时，出处必须经得起追问。

投资者关系 (IR)

典型痛点：季报/披露数据出处复杂，回答分析师与复核部门追问时，寻找原始凭证耗时耗力。

BriefLoop 价值：自动生成带溯源坐标的来源地图 (Source Map)，披露数据 30 秒回溯到原始页码与上下文。

战略与投资研究

典型痛点：周度竞品跟踪与行业简报改动频繁，多轮编辑后容易混入未经证实的数据观点。

BriefLoop 价值：核心论点先冻结再润色，多轮修改不会改变事实本质。

合规与审查部门

典型痛点：人工复核 AI 草稿的原始来源成本高昂，难以核准是否存在过期数据或缺失依据。

BriefLoop 价值：过期或无依据的陈述被自动标记并阻断，复核只看例外，不用全文重核。

管理层决策简报

典型痛点：多来源混合简报容易失真，AI 生成的判断结论过于绝对，缺乏归因支撑。

BriefLoop 价值：结论必须有登记来源支撑；最终放行的永远是人，且放行动作留有记录。

The Loop · 运行环

一条断言的完整旅程

从信源到人工放行，每个断言都在确定性控制面上留痕。门禁不放行的断言，回到修复回路，而不是溜进交付件。

有账可查的断言 → 交付 被门禁阻断 → 修复回路 实线节点 = 确定性控制面，虚线节点 = AI 起草
Quickstart · 上手

15 分钟跑通第一份简报

CLI 负责建档、校验和门禁；写作由你选择的 Agent runtime 完成，CLI 本身不会调用 LLM 写报告。

Start here 选择现在就能跑通的路径
01pipx install briefloop
02briefloop new industry-weekly ./my-weekly
03briefloop run --workspace ./my-weekly --runtime operator
01git clone https://github.com/Stahl-G/briefloop.git
02cd briefloop && bash scripts/setup.sh
03source .venv/bin/activate
04briefloop claude install --repo-workdir .

创建工作区和 operator handoff；CLI 本身不会调用 LLM 写报告。

安装后打开 Claude Code，输入 /briefloop new 开始对话式建档。

完整步骤见 15 分钟试运行指南；治理模型、门禁模拟与开发者细节见 运作原理。
